# 軟體需求規格書 (Software Requirements Specification)

## 專案概述

**專案名稱**：autoFetchStock - 台股即時資料抓取與視覺化系統

**專案描述**：一套以 Python 開發的台灣股票即時資料抓取與視覺化系統。使用者可透過輸入框輸入台股名稱或代號，系統自動從台灣證交所 (TWSE) API 抓取對應股票資料，並以分時走勢圖及 K 線圖等方式呈現。系統支援定時排程抓取、本地 JSON 檔案儲存，以及多種時間週期的技術分析圖表。

**技術選型**：Python

**資料來源**：台灣證交所 (TWSE) API

**儲存方式**：本地 JSON 檔案

---

## 功能性需求

### 核心系統需求

#### REQ-001 (基礎型 / Ubiquitous)
系統 (autoFetchStock) 應以 Python 程式語言實作所有功能模組。

#### REQ-002 (基礎型 / Ubiquitous)
系統應透過台灣證交所 (TWSE) API 作為唯一的股票資料來源。

#### REQ-003 (基礎型 / Ubiquitous)
系統應將所有抓取的股票資料以 JSON 格式儲存於本地檔案系統。

#### REQ-004 (基礎型 / Ubiquitous)
系統應提供圖形化使用者介面 (GUI)，包含股票搜尋輸入框、資訊顯示區域及雙頁籤圖表區域。

#### REQ-005 (基礎型 / Ubiquitous)
系統應採用台灣股市慣例，以紅色表示上漲、綠色表示下跌。

---

### 使用者輸入需求

#### REQ-010 (事件驅動型 / Event-Driven)
當使用者於輸入框中輸入台股名稱或股票代號時，系統應根據輸入內容自動查詢並抓取對應的台股資料。

#### REQ-011 (事件驅動型 / Event-Driven)
當使用者提交股票查詢請求時，系統應於 3 秒內回應並開始顯示該股票的基本資訊。

#### REQ-012 (事件驅動型 / Event-Driven)
當使用者輸入的股票名稱或代號存在多筆匹配結果時，系統應顯示匹配清單供使用者選擇。

---

### 股票資訊顯示需求

#### REQ-020 (事件驅動型 / Event-Driven)
當系統成功抓取股票資料後，系統應於介面最上方顯示該股票的名稱與股票代號。

#### REQ-021 (事件驅動型 / Event-Driven)
當系統成功抓取股票資料後，系統應於股票名稱與代號下方顯示以下即時資訊：
- 即時股價（最新成交價）
- 漲跌百分比（以百分比格式呈現）
- 漲跌幣值（以新台幣金額呈現）
- 總成交量（以「張」為單位呈現）

#### REQ-022 (狀態驅動型 / State-Driven)
當股價為上漲狀態時，系統應以紅色字體顯示股價、漲跌百分比及漲跌幣值。

#### REQ-023 (狀態驅動型 / State-Driven)
當股價為下跌狀態時，系統應以綠色字體顯示股價、跌幅百分比及跌幅幣值。

#### REQ-024 (狀態驅動型 / State-Driven)
當股價為平盤狀態時，系統應以白色（或中性色）字體顯示股價及相關資訊。

當使用者點選「分時資料」頁籤時，系統應切換顯示分時走勢圖表區域。

當使用者點選「K 線圖」頁籤時，系統應切換顯示 K 線圖與成交量圖表區域。

### Tab 1 - 分時資料需求

#### REQ-040 (基礎型 / Ubiquitous)
系統應於「分時資料」頁籤中顯示當日股價分時走勢折線圖。

#### REQ-041 (基礎型 / Ubiquitous)
系統應於「分時資料」頁籤中顯示即時成交量折線圖。

#### REQ-042 (基礎型 / Ubiquitous)
系統應於即時成交量折線圖中以零軸為基準線，向上方向表示買入交易量，向下方向表示賣出交易量。

#### REQ-043 (狀態驅動型 / State-Driven)
當台灣股市處於交易時段（09:00 - 13:30）時，系統應以 TWSE API 允許的最短查詢間隔持續更新分時資料。

#### REQ-044 (事件驅動型 / Event-Driven)
當系統收到新的分時資料時，系統應即時更新股價折線圖與成交量折線圖，不需使用者手動重新整理。

#### REQ-045 (基礎型 / Ubiquitous)
系統應於分時走勢折線圖中標示前一日收盤價作為基準線。

---

### Tab 2 - K 線圖與成交量圖需求

#### REQ-050 (基礎型 / Ubiquitous)
系統應於「K 線圖」頁籤中顯示 K 線蠟燭圖，以紅色蠟燭表示上漲（收盤價高於開盤價），綠色蠟燭表示下跌（收盤價低於開盤價）。

#### REQ-051 (基礎型 / Ubiquitous)
系統應於 K 線圖上疊加顯示以下移動平均線，且每條均線使用不同顏色：
- MA5（5 日移動平均線）
- MA10（10 日移動平均線）
- MA20（20 日移動平均線）
- MA60（60 日移動平均線）

#### REQ-052 (基礎型 / Ubiquitous)
系統應於 K 線圖上方或指定區域顯示 OHLC 資料，包含：
- 開盤價 (Open)
- 最高價 (High)
- 最低價 (Low)
- 收盤價 (Close)

#### REQ-053 (基礎型 / Ubiquitous)
系統應於 K 線圖下方顯示成交量柱狀圖，柱狀圖顏色應與對應 K 線蠟燭的漲跌顏色一致。

#### REQ-054 (基礎型 / Ubiquitous)
系統應於成交量柱狀圖上疊加顯示以下成交量均線：
- 均量 5（5 期成交量移動平均線）
- 均量 20（20 期成交量移動平均線）
- 均量 60（60 期成交量移動平均線）

#### REQ-055 (基礎型 / Ubiquitous)
系統應提供時間週期選擇器，支援以下週期切換：
- 日 K
- 週 K
- 月 K
- 1 分 K
- 5 分 K
- 15 分 K
- 30 分 K
- 60 分 K

#### REQ-056 (事件驅動型 / Event-Driven)
當使用者切換時間週期選擇器時，系統應於 2 秒內重新繪製對應週期的 K 線圖、均線及成交量圖。

#### REQ-057 (基礎型 / Ubiquitous)
系統應於 K 線圖區域中標示當前可視範圍內的最高價位與最低價位，並以文字標註具體數值。

#### REQ-058 (事件驅動型 / Event-Driven)
當使用者將滑鼠移至 K 線蠟燭上方時，系統應顯示該根 K 線的日期、開盤價、最高價、最低價、收盤價及成交量資訊。

### 資料抓取與排程需求

#### REQ-060 (基礎型 / Ubiquitous)
系統應實作定時排程機制，以 TWSE API 允許的最短查詢時間間隔自動抓取股票資料。

#### REQ-061 (狀態驅動型 / State-Driven)
當台灣股市處於交易時段（09:00 - 13:30）時，系統應依照排程機制自動執行資料抓取任務。

#### REQ-062 (狀態驅動型 / State-Driven)
當台灣股市處於非交易時段時，系統應停止自動排程抓取，僅允許使用者手動查詢歷史資料。

#### REQ-063 (事件驅動型 / Event-Driven)
當排程觸發資料抓取任務時，系統應向 TWSE API 發送查詢請求並將回傳的資料解析後儲存至本地 JSON 檔案。

#### REQ-064 (基礎型 / Ubiquitous)
系統應記錄每次資料抓取的時間戳記，確保資料的時序完整性。

---

### 資料儲存需求

#### REQ-070 (基礎型 / Ubiquitous)
系統應將每支股票的資料儲存為獨立的 JSON 檔案，檔案命名應包含股票代號。

#### REQ-071 (基礎型 / Ubiquitous)
系統應於 JSON 檔案中儲存以下欄位：日期、時間、開盤價、最高價、最低價、收盤價、成交量、成交金額。

#### REQ-072 (事件驅動型 / Event-Driven)
當新資料寫入 JSON 檔案時，系統應以追加方式寫入，不覆蓋既有歷史資料。

#### REQ-073 (事件驅動型 / Event-Driven)
當系統啟動時，系統應自動載入本地 JSON 檔案中的既有歷史資料。

## 錯誤處理

#### REQ-100 (異常行為型 / Unwanted Behavior)
若 TWSE API 連線逾時（超過 10 秒無回應），則系統應中斷該次請求，顯示「網路連線逾時，請稍後再試」錯誤訊息，並於 30 秒後自動重試一次。

#### REQ-101 (異常行為型 / Unwanted Behavior)
若 TWSE API 回傳非預期的資料格式或空資料，則系統應捨棄該次回傳資料，記錄錯誤日誌，並於介面顯示「資料格式異常，已略過本次更新」警告訊息。

#### REQ-102 (異常行為型 / Unwanted Behavior)
若使用者輸入的股票代號或名稱不存在於 TWSE 資料庫中，則系統應顯示「查無此股票，請確認輸入內容」錯誤訊息。

#### REQ-103 (異常行為型 / Unwanted Behavior)
若本地 JSON 檔案讀取失敗（檔案損毀或格式錯誤），則系統應備份損毀檔案、建立新的空白 JSON 檔案，並顯示「歷史資料載入失敗，已重新建立資料檔」警告訊息。

#### REQ-104 (異常行為型 / Unwanted Behavior)
若系統偵測到 TWSE API 連續 3 次請求失敗，則系統應暫停自動排程抓取，並於介面顯示「資料來源暫時無法存取，自動更新已暫停」警告訊息。

#### REQ-105 (異常行為型 / Unwanted Behavior)
若系統磁碟空間不足（剩餘空間低於 100 MB），則系統應停止資料寫入操作，並顯示「磁碟空間不足，請清理磁碟後重試」錯誤訊息。

#### REQ-106 (異常行為型 / Unwanted Behavior)
若排程任務執行過程中發生未預期的例外錯誤，則系統應捕捉該例外、記錄完整錯誤堆疊至日誌檔案，並繼續執行下一次排程而不中斷系統運作。
